# 📁 Simple Secure File Server (Python)

Минималистичный, но функциональный файловый сервер на Python с авторизацией, загрузкой файлов, управлением сессиями и входом через QR-код с подтверждением администратора.

Проект ориентирован на быстрый разворот локального сервера для обмена файлами без лишних зависимостей и сложной настройки.

---

## 🚀 Возможности

- 🔐 Авторизация пользователей (обычный пользователь и администратор)
- 🍪 Сессионная система с автоматическим продлением
- 📂 Удобный просмотр файлов и папок через браузер
- ⬇️ Скачивание файлов одним кликом
- ⬆️ Загрузка файлов через форму или drag & drop
- 📦 Автоматическая упаковка загружаемых файлов в ZIP
- 📱 Вход через QR-код с подтверждением администратора
- 🛡 Защита файловой системы от выхода за пределы рабочей директории
- 💾 Сохранение активных сессий между перезапусками
- 🎨 Встроенный тёмный интерфейс без внешних библиотек

---

## ⚙️ Установка и запуск

### 1. Клонирование

```bash
git clone https://github.com/your-repo/file-server.git
cd file-server
````

### 2. Установка зависимостей

```bash
pip install qrcode[pil]
```

### 3. Запуск

```bash
python server.py
```

Открыть в браузере:

```
http://localhost:6574
```

---

## 🪟 Запуск в фоне (Windows / PowerShell)

Можно запустить сервер скрыто, без окна консоли:

```powershell
powershell -NoProfile -Command "Start-Process -FilePath 'C:\Windows\py.exe' -ArgumentList 'B:\frp\sec_server.py' -WorkingDirectory 'B:\dz' -WindowStyle Hidden"
```

### Что это делает:

* запускает Python-скрипт через `py.exe`
* скрывает окно (`Hidden`)
* задаёт рабочую директорию
* не использует профиль PowerShell (быстрее старт)

Удобно для:

* автозапуска
* фонового сервера
* использования вместе с планировщиком задач

## 🧾 Конфигурация

Все основные параметры можно задать через `.env` файл:

```env

ADMIN_NAME=admin

PORT=6574
```

Если файл отсутствует — используются значения по умолчанию.

## 🔐 Авторизация и роли

Система поддерживает два уровня доступа:

### 👤 Пользователь

* Может просматривать файлы
* Может скачивать файлы
* Может загружать файлы
* Сессия ограничена по времени

### 👑 Администратор

* Все возможности пользователя
* Подтверждение входа через QR
* Постоянная (не истекающая) сессия

---

## 🍪 Сессии

* Сессии хранятся в `sessions.json`
* Для обычных пользователей:

  * время жизни ~1 час 40 минут
  * автоматически продлевается при активности
* Для администратора:

  * постоянная сессия без ограничения времени
* Используются безопасные cookies:

  * `HttpOnly`
  * `SameSite=Lax`

---

## 📱 Вход через QR-код

Одна из ключевых функций проекта.

### Как работает:

1. Пользователь открывает страницу `/qr-login`
2. Сервер генерирует временный токен (живёт 60 секунд)
3. Отображается QR-код с ссылкой подтверждения
4. Администратор сканирует QR-код
5. Администратор подтверждает или отклоняет вход
6. При подтверждении пользователь автоматически авторизуется

### Особенности:

* Привязка к IP и User-Agent
* Одноразовый токен
* Защита от повторного использования
* Автоматическое истечение

---

## 📦 Загрузка файлов

* Поддерживается загрузка через:

  * стандартную форму
  * drag & drop
* Максимальный размер файла: **4 ГБ**
* Каждый файл автоматически:

  * упаковывается в ZIP
  * получает имя по timestamp
* Все файлы сохраняются в папку `upload/`

---

## 📂 Работа с файлами

* Сервер отображает текущую директорию
* Поддерживается:

  * переход по папкам
  * возврат назад
* Файлы:

  * скачиваются через `Content-Disposition`
* Папки:

  * отображаются отдельно с иконками

---

## 🛡 Безопасность

* Проверка путей (защита от directory traversal)
* Ограничение размера загружаемых файлов
* Очистка просроченных сессий
* Очистка просроченных QR-токенов
* HTML-экранирование вывода
* Cookie без доступа из JavaScript

---

## 🧠 Особенности реализации

* Используется стандартный `http.server`
* Минимум зависимостей
* Полностью самодостаточный сервер
* Нет необходимости в базе данных
* Хранение состояния через JSON
